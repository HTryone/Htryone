# Typora 笔记索引生成器 - 用户使用手册

## 1. 产品简介

Typora 笔记索引生成器是一款帮助用户管理 Markdown 笔记的工具。它能够自动扫描指定目录，生成包含文件夹结构图和可点击目录列表的索引文件，让用户快速浏览和定位笔记。

### 1.1 主要特性
- ✅ 自动扫描多种文件格式（md/doc/docx/ppt/pptx/html/xls/xlsx/csv/pdf）
- ✅ 生成 ASCII 艺术风格的文件夹结构图
- ✅ 生成带层级缩进的可点击目录列表
- ✅ 支持自动监控文件变化，实时更新索引
- ✅ 跨平台支持（Windows/macOS/Linux）
- ✅ 零依赖运行（仅核心功能）

## 2. 安装指南

### 2.1 环境要求
- **操作系统**：Windows 10/11、macOS 10.14+、Linux
- **Python 版本**：Python 3.7 或更高版本
- **可选依赖**：`watchdog`（用于文件监控功能）

### 2.2 安装步骤

#### 步骤 1：检查 Python 环境
打开终端/命令提示符，输入以下命令：

```bash
python --version
```

如果显示 Python 3.7+ 的版本号，说明环境已就绪。

> 💡 **没有 Python？** 请访问 https://www.python.org/downloads/ 下载安装。

#### 步骤 2：下载脚本
将 `update_index.py` 和 `monitor.py` 保存到你的笔记根目录。

> 💡 **提示**：建议将脚本放在笔记库的根目录，这样最方便使用。

#### 步骤 3：（可选）安装监控依赖
如果需要使用自动监控功能，安装 watchdog：

```bash
pip install watchdog
```

## 3. 快速开始

### 3.1 一键生成索引

1. 打开终端，进入笔记目录：
   ```bash
   cd D:\个人创作
   ```

2. 运行脚本：
   ```bash
   python update_index.py
   ```

3. 等待几秒，会看到以下输出：
   ```
   [INFO] 正在扫描目录：D:\个人创作
   [OK] 索引生成成功！
   [OK] 文件位置：D:\个人创作\_Index.md
   [OK] 用 Typora 打开即可查看树状图和目录。
   ```

4. 用 Typora 打开生成的 `_Index.md` 文件即可。

### 3.2 启用自动监控（可选）

如果你希望在修改文件后自动更新索引，运行监控程序：

```bash
python monitor.py
```

终端会显示：
```
👀 监控已启动，正在监视笔记目录...
💡 提示：支持格式文件的增删改都会自动刷新索引。
   支持格式：.md, .doc, .docx, .ppt, .pptx, .html, .htm, .xls, .xlsx, .csv, .pdf
   按 Ctrl+C 可停止监控。
```

当你添加、修改或删除支持的文件时，索引会自动重新生成。

## 4. 配置指南

### 4.1 修改配置

用文本编辑器打开 `update_index.py`，找到顶部的「配置区」：

```python
# ==================== 配置区（可按需修改） ====================
NOTES_DIR = "."                                          # 笔记根目录
OUTPUT_FILE = "_Index.md"                                # 生成的索引文件名
IGNORE_DIRS = {".git", ".obsidian", "assets", "images", 
               "附件", "__pycache__", "templates", "build"}  # 忽略的文件夹
SCAN_EXTENSIONS = {".md", ".doc", ".docx", ".ppt", ".pptx", 
                   ".html", ".htm", ".xls", ".xlsx", ".csv", ".pdf"}  # 扫描的文件类型
# ============================================================
```

### 4.2 配置项说明

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `NOTES_DIR` | `"."` | 笔记根目录，`.` 表示脚本所在目录 |
| `OUTPUT_FILE` | `"_Index.md"` | 生成的索引文件名 |
| `IGNORE_DIRS` | 多个目录 | 扫描时忽略的文件夹名称 |
| `SCAN_EXTENSIONS` | 多种扩展名 | 需要扫描的文件类型 |

### 4.3 常见配置场景

#### 场景 1：扫描其他目录
```python
NOTES_DIR = "D:/我的文档/笔记"
```

#### 场景 2：忽略更多目录
```python
IGNORE_DIRS = {
    ".git", ".obsidian", "assets", "images",
    "附件", "__pycache__", "templates", "build",
    "node_modules",  # 新增：忽略 node_modules
    ".vscode"        # 新增：忽略 VSCode 配置
}
```

#### 场景 3：只扫描 Markdown 文件
```python
SCAN_EXTENSIONS = {".md"}
```

## 5. 使用技巧

### 5.1 创建快捷方式（Windows）

为了方便使用，可以创建批处理文件：

1. 在脚本目录创建 `生成索引.bat`：
   ```batch
   @echo off
   python update_index.py
   pause
   ```

2. 双击即可运行。

### 5.2 开机自启监控

如果你希望开机后自动监控笔记目录：

1. 按 `Win + R`，输入 `shell:startup`，回车
2. 在打开的文件夹中，创建 `笔记监控.bat`：
   ```batch
   @echo off
   cd /d D:\个人创作
   python monitor.py
   ```
3. 保存后关闭

### 5.3 配合 Typora 使用

1. **打开索引文件**：双击 `_Index.md` 用 Typora 打开
2. **按住 Ctrl 点击链接**：跳转到对应文件
3. **固定索引文件**：在 Typora 中勾选「固定到侧边栏」，方便随时查看

### 5.4 目录结构建议

建议采用清晰的目录结构，例如：
```
D:\个人创作\
├── 工作/
│   ├── 项目A/
│   └── 项目B/
├── 学习/
│   ├── 编程/
│   └── 语言/
└── 生活/
    └── 日记/
```

## 6. 故障排除

### 6.1 常见问题

#### Q1：运行时显示"不是内部或外部命令"
**原因**：Python 未添加到 PATH  
**解决**：
1. 重新安装 Python，勾选 "Add Python to PATH"
2. 或在命令提示符中使用完整路径：`C:\Python39\python.exe update_index.py`

#### Q2：运行时显示"No module named 'xxx'"
**原因**：缺少 watchdog 模块（仅使用 monitor.py 时）  
**解决**：
```bash
pip install watchdog
```

#### Q3：中文文件名显示乱码
**原因**：文件编码问题  
**解决**：确保脚本和输出文件都使用 UTF-8 编码。`update_index.py` 已默认使用 UTF-8。

#### Q4：索引文件没有更新
**原因**：监控程序未运行，或文件不在支持列表中  
**解决**：
1. 确保 `monitor.py` 正在运行
2. 检查文件扩展名是否在 `SCAN_EXTENSIONS` 中
3. 检查文件所在目录是否在 `IGNORE_DIRS` 中

### 6.2 错误信息说明

| 错误信息 | 含义 |
|---------|------|
| `[WARN] 未找到任何支持的文件。` | 目录中没有匹配的文件，或都在忽略列表中 |
| `[ERROR] 目录不存在：xxx` | 指定的笔记目录路径错误 |
| `UnicodeEncodeError` | 控制台不支持中文/特殊字符（Windows 旧版） |

## 7. 卸载指南

### 7.1 完全卸载
1. 删除 `update_index.py`
2. 删除 `monitor.py`
3. 删除 `_Index.md`（如果不再需要）
4. 删除启动项中的批处理文件（如果创建过）

### 7.2 保留数据
- 删除脚本后，你的笔记文件不受影响
- `_Index.md` 可以保留作为手动参考
