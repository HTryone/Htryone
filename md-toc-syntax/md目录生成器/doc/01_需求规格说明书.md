# Typora 笔记索引生成器 - 需求规格说明书

## 1. 项目概述

### 1.1 项目名称
**Typora 笔记索引生成器**（以下简称"索引器"）

### 1.2 项目背景
随着笔记数量和层级的增加，用户需要一个自动化工具，能够：
- 快速生成笔记目录索引
- 可视化展示文件夹结构
- 支持多种文件格式
- 保持索引与文件系统同步

### 1.3 项目目标
为 Typora 用户提供一个自动化索引生成工具，实现：
- 一键扫描并生成 Markdown 格式的索引文件
- 支持多层级目录结构的可视化展示
- 支持多种常见文档格式
- 可选的文件监控功能，实时同步更新

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 文件扫描
- **扫描范围**：指定根目录及其所有子目录
- **递归扫描**：支持无限层级子目录
- **文件过滤**：仅扫描支持的文件格式
- **目录过滤**：可配置忽略特定目录（如 `.git`、`assets` 等）

#### 2.1.2 文件格式支持
| 格式 | 扩展名 | 图标 | 说明 |
|------|--------|------|------|
| Markdown | `.md` | 📝 | 主要支持的笔记格式 |
| Word 文档 | `.doc`, `.docx` | 📘 | 微软 Word 文档 |
| PPT 演示 | `.ppt`, `.pptx` | 📊 | 微软 PowerPoint 演示文稿 |
| HTML 网页 | `.html`, `.htm` | 🌐 | 网页文件 |
| Excel 表格 | `.xls`, `.xlsx`, `.csv` | 📈 | 微软 Excel 表格 |
| PDF 文档 | `.pdf` | 📕 | 便携式文档格式 |

#### 2.1.3 索引生成
- **输出格式**：标准 Markdown 文件（`_Index.md`）
- **内容包含**：
  - 文档标题和元信息（更新时间、文件总数）
  - 文件夹结构树状图（ASCII 艺术风格）
  - 可点击的目录列表（按层级展示）
- **链接生成**：使用相对路径或绝对路径生成可点击链接

#### 2.1.4 文件夹结构展示
- **树状图**：ASCII 艺术风格的文件夹结构可视化
  - 目录图标：`📁`
  - 连接线：`├──`、`└──`、`│`
  - 缩进：4 空格
- **目录列表**：带缩进的层级列表
  - 主文件夹：H3 标题
  - 子文件夹/文件：H4 标题 + 树形缩进符号
  - 缩进符号：`├─`（非末项）、`└─`（末项）、`│`（垂直连接线）

### 2.2 扩展功能

#### 2.2.1 文件监控（可选）
- **监控方式**：使用 `watchdog` 库监控文件系统变化
- **触发条件**：文件的创建、修改、删除、移动
- **响应动作**：自动重新生成索引
- **限流机制**：避免短时间内多次触发

### 2.3 用户配置

#### 2.3.1 可配置项
```python
NOTES_DIR = "."              # 笔记根目录
OUTPUT_FILE = "_Index.md"   # 输出文件名
IGNORE_DIRS = {...}         # 忽略的目录列表
SCAN_EXTENSIONS = {...}     # 扫描的文件扩展名
```

#### 2.3.2 默认配置
- **忽略目录**：`.git`, `.obsidian`, `assets`, `images`, `附件`, `__pycache__`, `templates`, `build`
- **输出文件**：`_Index.md`（生成在笔记根目录下）

## 3. 非功能需求

### 3.1 性能需求
- 扫描 1000 个文件应在 5 秒内完成
- 生成的索引文件应小于 1MB

### 3.2 兼容性需求
- 支持 Windows/macOS/Linux 多平台
- 兼容 Python 3.7 及以上版本
- 生成的 Markdown 兼容 Typora、VS Code 等主流编辑器

### 3.3 可维护性需求
- 代码结构清晰，模块化设计
- 配置文件与逻辑代码分离
- 注释完整，易于理解和修改

## 4. 用例分析

### 4.1 用例 1：手动生成索引
**参与者**：用户  
**前置条件**：用户已安装 Python 3.7+  
**主流程**：
1. 用户双击运行 `update_index.py`
2. 系统扫描笔记目录
3. 系统生成 `_Index.md` 文件
4. 用户用 Typora 打开索引文件

**备选流程**：
- 无支持文件时，显示警告信息
- 文件夹为空时，生成空索引（带提示）

### 4.2 用例 2：自动监控更新
**参与者**：用户  
**前置条件**：用户已安装 Python 3.7+ 和 watchdog 库  
**主流程**：
1. 用户运行 `monitor.py`
2. 系统启动文件监控
3. 用户对支持的文件进行增删改操作
4. 系统自动检测变化并重新生成索引
5. 用户按 Ctrl+C 停止监控

### 4.3 用例 3：自定义配置
**参与者**：用户  
**前置条件**：用户具备基础 Python 知识  
**主流程**：
1. 用户打开 `update_index.py`
2. 用户修改配置区的参数
3. 用户保存文件并重新运行

## 5. 验收标准

### 5.1 功能验收
- [ ] 脚本能够正确扫描指定目录下的所有支持格式文件
- [ ] 生成的索引文件包含完整的文件夹结构图
- [ ] 目录列表正确显示层级关系
- [ ] 文件链接可点击并正确跳转
- [ ] 忽略目录被正确排除

### 5.2 格式验收
- [ ] 树状图使用标准 ASCII 字符，视觉整齐
- [ ] 目录列表缩进符号使用 `├─` `└─` `│`
- [ ] 不同文件类型显示对应图标
- [ ] 主文件夹使用 H3，子内容使用 H4
