## 2. 核心算法设计

### 2.1 文件扫描算法
**算法名称**：递归目录遍历 + 文件过滤

**输入**：
- 根目录路径
- 支持的文件扩展名集合
- 忽略的目录名称集合

**输出**：
- 支持文件的 `Path` 对象列表

**算法流程**：
```
1. 初始化空列表 md_files
2. 使用 rglob("*") 递归遍历目录
3. 对每个路径 p:
   a. 如果 p 不是文件，跳过
   b. 如果 p 的路径部分包含忽略目录名，跳过
   c. 如果 p 是输出文件本身 (_Index.md)，跳过
   d. 如果 p 的扩展名不在支持列表中，跳过
   e. 否则，将 p 加入 md_files 列表
4. 返回 md_files
```

**时间复杂度**：O(n)，n 为文件总数

### 2.2 文件夹树构建算法
**算法名称**：前缀树（Trie）结构构建

**输入**：文件列表
**输出**：嵌套字典结构

**数据结构设计**：
```python
# 示例结构
{
    "know": {
        "hu共享": {
            "个人简历": {
                "260405HU.docx": Path("D:/.../260405HU.docx"),
                "面试回答.md": Path("D:/.../面试回答.md")
            },
            "模板": {
                "通用模板.docx": Path("D:/.../通用模板.docx")
            }
        },
        "知识库": {...}
    },
    "笔记": {
        "README.md": Path("D:/.../README.md")
    }
}
```

**算法流程**：
```
1. 初始化空字典 root
2. 对每个文件 file:
   a. 获取相对路径: rel = file.relative_to(base_path)
   b. 拆分为路径部分: parts = list(rel.parts)
   c. 从根开始，逐层构建字典:
      - 如果是最后一部分，存入 Path 对象
      - 否则，在当前位置创建子字典
3. 返回 root
```

### 2.3 ASCII 树状图渲染算法
**输入**：嵌套字典结构
**输出**：ASCII 艺术风格的树状图

**核心逻辑**：
```python
def render_tree_ascii(tree, prefix="", is_last=True):
    lines = []
    items = 排序(目录在前，文件在后)
    
    for i, (name, content) in enumerate(items):
        is_last_item = (i == len(items) - 1)
        connector = "└── " if is_last_item else "├── "
        
        if content is None:
            # 渲染文件
            lines.append(f"{prefix}{connector}{图标} {name}")
        else:
            # 渲染目录
            lines.append(f"{prefix}{connector}{图标} {name}/")
            # 递归渲染子目录
            new_prefix = prefix + ("    " if is_last_item else "│   ")
            lines.extend(render_tree_ascii(content, new_prefix, is_last_item))
    
    return lines
```

**连接线规则**：
| 情况 | 连接线 |
|------|--------|
| 非最后一项 | `├── ` |
| 最后一项 | `└── ` |
| 非最后项的子级缩进 | `│   ` |
| 最后项的子级缩进 | `    ` (4空格) |

### 2.4 目录列表渲染算法
**输入**：嵌套字典结构
**输出**：带标题和缩进的 Markdown

**层级规则**：
| 层级 | 深度 | 格式 | 示例 |
|------|------|------|------|
| 主文件夹 | depth=0 | H2 标题，无缩进 | `## 📁 know/ （19 篇）` |
| 子文件夹 | depth>=1 | 缩进列表 | `  - 📁 **know/hu共享/**` |
| 更深文件夹 | depth>=2 | 更多缩进 | `    - 📁 **know/hu共享/个人简历/**` |
| 文件 | 任意 | 缩进列表 | `      - 📘 [文件名.docx](链接)` |

**核心逻辑**：
```python
def render_folder_list(tree, base_path="", depth=0):
    lines = []
    items = 排序(目录在前，文件在后)

    for name, content in items:
        if isinstance(content, dict):
            file_count = count_files_in_tree(content)
            full_path = f"{base_path}{name}" if base_path else name

            if depth == 0:
                lines.append(f"## 📁 {full_path}/ （{file_count} 篇）")  # H2 无缩进
            else:
                indent = "  " * depth
                lines.append(f"{indent}- 📁 **{full_path}/** （{file_count} 篇）")  # 缩进列表

            sub_path = f"{full_path}/"
            lines.extend(render_folder_list(content, sub_path, depth + 1))
        else:
            indent = "  " * max(1, depth)
            lines.append(f"{indent}- {图标} [{文件名}]({链接})")

    return lines
```

### 3.3 树节点结构
```python
# 文件夹树节点：字典
{
    "文件夹名": {
        "子文件夹": {...},
        "文件.md": Path("D:/.../文件.md")
    }
}

# 递归统计文件数
def count_files_in_tree(node):
    if isinstance(node, Path):
        return 1
    return sum(count_files_in_tree(v) for v in node.values())
```

## 4. 接口设计

### 4.1 主程序接口

#### `generate_md_index(base_dir: str) -> None`
**功能**：生成 Markdown 索引文件  
**参数**：
- `base_dir`: 笔记根目录路径（字符串）

**返回值**：无，直接生成 `_Index.md` 文件

**执行流程**：
1. 解析并验证目录路径
2. 扫描并收集支持的文件
3. 构建文件夹树结构
4. 渲染 ASCII 树状图
5. 渲染目录列表
6. 写入输出文件

#### `build_folder_tree(files: List[Path], base_path: Path) -> dict`
**功能**：构建嵌套字典树结构  
**参数**：
- `files`: 文件 Path 对象列表
- `base_path`: 根目录 Path 对象

**返回值**：嵌套字典结构，文件节点为 Path 对象，目录节点为 dict

#### `render_tree_ascii(tree: dict, prefix: str, is_last: bool) -> List[str]`
**功能**：递归渲染 ASCII 树状图  
**参数**：
- `tree`: 嵌套字典结构
- `prefix`: 当前前缀字符串
- `is_last`: 是否为父级的最后一项

**返回值**：行字符串列表

#### `render_folder_list(tree: dict, base_path: str, depth: int) -> List[str]`
**功能**：递归渲染带缩进的目录列表  
**参数**：
- `tree`: 嵌套字典结构
- `base_path`: 当前路径前缀（用于显示完整路径）
- `depth`: 当前深度（从 0 开始）

**返回值**：行字符串列表

### 4.2 监控程序接口

#### `IndexHandler(FileSystemEventHandler)`
**功能**：文件系统事件处理器  
**继承**：`watchdog.events.FileSystemEventHandler`

**方法**：
- `on_any_event(event)`: 处理任意文件系统事件

**触发条件**：
- `event.event_type` in `['modified', 'created', 'deleted', 'moved']`
- 文件扩展名在 `SCAN_EXTENSIONS` 中

## 7. 错误处理设计

### 7.1 异常类型
| 异常情况 | 处理方式 | 用户提示 |
|---------|---------|---------|
| 目录不存在 | 打印警告，退出 | `[ERROR] 目录不存在：` |
| 无支持文件 | 打印警告，退出 | `[WARN] 未找到任何支持的文件。` |
| 文件权限不足 | 跳过该文件，继续 | `[WARN] 无法访问：xxx` |
| 编码错误 | 使用 UTF-8，写入失败时提示 | `[ERROR] 文件写入失败` |

### 7.2 日志级别
| 级别 | 前缀 | 场景 |
|------|------|------|
| INFO | `[INFO]` | 正常流程信息 |
| OK | `[OK]` | 操作成功提示 |
| WARN | `[WARN]` | 非致命问题提示 |
| ERROR | `[ERROR]` | 致命错误提示 |

## 9. 代码优化记录

### 9.1 v1.1 优化（2026-04-21）
1. **去重 count_files 函数**：合并多处重复定义为顶层函数 `count_files_in_tree()`
2. **图标映射表**：将 `get_file_icon()` 改为字典查找 `FILE_ICONS.get()`

**优化前代码结构**：
- `build_tree_dict()` - 构建树（重复逻辑）
- `render_tree_dict()` - ASCII 树状图
- `render_folder_tree_simple()` - 主文件夹渲染
- `render_sub_content()` - 子内容渲染（含重复 count_files）
- 多处重复的 `count_files()` 内联函数

**优化后代码结构**：
- `build_folder_tree()` - 统一构建树结构
- `convert_to_ascii_tree()` - 转换为 ASCII 树需要的结构
- `render_tree_ascii()` - ASCII 树状图
- `render_folder_list()` - 统一渲染目录列表
- `count_files_in_tree()` - 顶层函数，递归统计
