## 1 前言

### 1.1 基本原则

1. 纯 Markdown 优先：尽量使用标准语法，减少 HTML 标签
2. Typora 兼容：避免 Typora 渲染异常的语法
3. 可读性优先：源码本身也要易读，不只是渲染后
4. 跳转可用：支持页内跳转、文件跳转、外部链接

---

## 2 禁止与允许的语法

### 2.1 引用块做信息展示

**问题**：Typora 中显示为下箭头样式，不美观

**禁止**（用引用块展示键值信息）：
```markdown
> 版本：1.0
> 最后更新：2026-04-23
```

**推荐**：
```markdown
**版本**：1.0
**最后更新**：2026-04-23
```

> **注**：引用块本身语法不禁止。引用他人文字、标注来源时可以正常使用 `>`。

---

### 2.2 行尾双空格

**问题**：Markdown 软换行语法，Typora 可能显示异常

**禁止**（行尾有两个空格）：
```markdown
第一行  
第二行
```

**推荐**：
```markdown
第一行

第二行
```

---

### 2.3 HTML 标签策略

Typora 支持部分 HTML 标签，按**简单/复杂**区分：

**✅ 允许的简单 HTML**：

| 标签 | 用途 | 示例 |
|---|---|---|
| `<u>` | 下划线 | `<u>下划线文字</u>` |
| `<br>` | 强制换行 | `文字<br>换行` |
| `<font>` | 字体颜色/大小 | `<font color="red">红色</font>` |
| `<p align>` | 对齐 | `<p align="center">居中</p>` |
| `<a id>` | 自定义锚点 | `<a id="note"></a>` |
| `<img>` | 插入图片（控制尺寸） | `<img src="x.png" width="100"/>` |
| `<!-- -->` | 注释 | `<!-- 不可见 -->` |

**❌ 禁止的复杂 HTML**：

| 标签 | 问题 |
|---|---|
| `<details>` / `<summary>` | Typora 不支持折叠块 |
| `<div style="...">` | 内联样式渲染不一致 |
| `<script>` | 安全风险，不应出现在文档中 |
| `<iframe>` | Typora 不支持嵌入外部页面 |
| `<table>` (HTML) | 用 Markdown 表格语法替代 |

**原则**：能不用 HTML 就不用；必须用时，只用上表允许的简单标签。

---

### 2.4 其他禁止项

| 禁止项 | 原因 | 替代方案 |
|---|---|---|
| 行内样式 `style="..."` | 渲染不一致 | 用 Markdown 语法或 `<font>` 标签 |
| 嵌套 HTML 结构 | 难以维护 | 用 Markdown 标题 + 列表组织 |
| JavaScript 代码 | 安全风险 | 不使用 |

## 3 推荐的语法

### 3.1 文档头部

```markdown
文档标题

**版本**：1.0
**作者**：张三
**日期**：2026-04-23

---
```

---

### 3.2 章节结构

```markdown
一级标题

二级标题（大章节）

三级标题（小章节）

四级标题（细分）
```

---

### 3.3 字体样式

| 效果 | 语法 | 快捷键 |
|---|:---|:---|
| 粗体 | `**文字**` | Ctrl+B |
| 斜体 | `*文字*` | Ctrl+I |
| 粗斜体 | `***文字***` | Ctrl+B + Ctrl+I |
| 删除线 | `~~文字~~` | Alt+Shift+5 |
| 下划线 | `<u>文字</u>` | Ctrl+U |
| 行内代码 | `` `代码` `` | Ctrl+Shift+` |
| 高亮 | `==文字==` | - |
| 字体颜色 | `<font color="red">文字</font>` | - |
| 字体大小 | `<font size=4>文字</font>` | - |

---

### 3.4 列表

**无序列表**：
```markdown
- 项目一
- 项目二
  - 子项目（Typora 中按 Tab 缩进）
  - 子项目
```

**有序列表**：
```markdown
1. 第一步
2. 第二步
3. 第三步
```

**嵌套列表**（无序 + 有序混排）：
```markdown
- 无序第一项
  1. 有序子项一
  2. 有序子项二
- 无序第二项
```

**任务列表**（Typora 支持）：
```markdown
- [x] 已完成任务
- [ ] 未完成任务
```

> **Typora 技巧**：输入 `- ` 后回车自动续项，按 Tab 缩进变子项，按 Shift+Tab 取消缩进，连按两次回车退出列表。

---

### 3.5 表格

```markdown
| 列1 | 列2 | 列3 |
|---|:---:|---:|
| 居左 | 居中 | 居右 |
| 内容 | 内容 | 内容 |
```

**对齐方式**：
- `|---|` 居左（默认）
- `|:---:|` 居中对齐
- `|---:|` 右对齐

> **Typora 技巧**：Ctrl+T 快速插入表格，选中单元格后左上角可设置对齐方式。

---

### 3.6 代码块

**行内代码**：
```markdown
使用 `print()` 函数
```

**代码块**（指定语言后 Typora 会语法高亮）：
````markdown
```python
def hello():
    print("Hello")
```
````

**常用语言标识**：`python` `javascript` `typescript` `java` `c` `cpp` `go` `rust` `bash` `powershell` `sql` `json` `yaml` `html` `css` `markdown`

> **注意**：不指定语言时代码块全灰无高亮。务必加上语言标识。

---

### 3.7 分隔线

三种写法效果相同，推荐 `---`（最直观）：
```markdown
---
```

> **注意**：`---` 上下需要空行，否则可能被解析为标题。

## 4 跳转语法

### 4.1 外部链接

```markdown
[链接文字](https://www.example.com)

[带标题的链接](https://www.example.com "鼠标悬浮显示")
```

---

### 4.2 本地文件跳转

```markdown
[打开同级文件](./LinkTest.md)

[打开子目录图片](./img/LinkTest.png)

[打开上级目录文件](../config.md)
```

**路径规则**：
- `.` 一个小圆点表示当前目录，故 `./LinkTest.md` 表示当前目录下的 LinkTest.md 文件，`./img/LinkTest.png` 表示当前目录下的 img 文件夹下的 LinkTest.png 文件
- `..` 两个小圆点表示上一级目录

---

### 4.3 页内标题跳转

语法：`[链接文字](#标题文字)`

```markdown
[跳转到第一章第一节](#1.1 Markdown介绍)

[跳转到概述](#概述)

[跳转到安装步骤](#安装步骤)
```

**规则**：将标题文字原样复制到 `#` 后面的括号里，不做任何转换
1. 保留 emoji（📌🧱⚙️🏁 等）
2. 保留中文和中文标点（一、、：？等）
3. 保留空格，不替换为 `-`
4. 固定用一个 `#`，和标题级别无关

**示例**：
- 标题 `### 📌 一、Git 是什么？` → 锚点 `[Git 是什么？](#📌 一、Git 是什么？)`
- 标题 `### 1.1 Markdown介绍` → 锚点 `[跳转](#1.1 Markdown介绍)`

---

### 4.4 页内锚点跳转

**定义锚点**：
```markdown
<a id="important-note"></a>
```

**跳转到锚点**：
```markdown
[返回重要提示](#important-note)
```

---

### 4.5 参考链接（多处引用）

适用于同一链接在文档中多次使用：
```markdown
[官网][typora]
[文档][typora]

[typora]: https://typora.io "Typora 官网"
```

## 5 数学公式

### 5.1 行内公式与行间公式

**行内公式**（与文字在同一行）：
```markdown
公式 $E=mc^2$ 是质能方程
```

**行间公式**（独占一行，居中显示）：
```markdown
$$
E = mc^2
$$
```

> **Typora 设置**：需在 偏好设置 → Markdown → 开启「内联公式」和「上标/下标」选项。

---

## 6 Typora 特有功能

### 6.1 目录生成

```markdown
[TOC]
```

Typora 会自动根据标题生成目录，放在文档任意位置即可。

---

### 6.2 表情符号

```markdown
:smile: :heart: :rocket:
```

效果：😄 ❤️ 🚀

---

### 6.3 上标和下标

```markdown
上标：X^2^
下标：H~2~O
```

效果：X² H₂O

---

### 6.4 高亮

```markdown
==高亮文字==
```

效果：<mark>高亮文字</mark>（需在偏好设置中开启）

---

### 6.5 注释

```markdown
<!-- 这是注释，不会显示在渲染结果中 -->
```

---

### 6.6 图片

```markdown
![图片描述](图片路径.png)

![图片描述](图片路径.png "鼠标悬浮标题")
```

**注意**：使用相对路径，便于移植。

> **Typora 图片设置技巧**：点击 文件 → 偏好设置 → 图像，建议：
> - 插入图片时：复制到指定路径 `./img`
> - 勾选「对本地位置的图片应用上述规则」「对网络位置的图片应用上述规则」「自动上传」
> - 这样粘贴图片时 Typora 会自动复制到 `./img` 文件夹，文档移植后图片不会丢失

## 8 常见问题 FAQ

以下是日常使用中常见的问题和解答。遇到问题时先来这里看看。

---

### 8.1 我的表格怎么对不齐？

**同学**：在吗？我的表格显示乱七八糟的，好难受！

**管理员**：别急，先让我看看你的代码。你是不是这样写的？
```markdown
| 姓名 | 年龄
|---|---|
| 张三 | 20
```

**同学**：对对对！就是这样！

**管理员**：找到问题了。你的表头有 2 列，但每行的 `|` 数量不一致。表头、分隔线、内容行的列数必须相同：
```markdown
| 姓名 | 年龄 |
|---|---|
| 张三 | 20 |
```

**同学**：好了！谢谢！

---

### 8.2 页内跳转点了没反应？

**同学**：我写了 `[跳转](#安装步骤)`，但是点了没反应！

**管理员**：你的标题是怎么写的？

**同学**：`### 安装步骤`

**管理员**：锚点的规则是**原样复制标题文字**到 `#` 后面，不做任何转换。你的标题是 `### 安装步骤`，那锚点就写：
```markdown
[跳转到安装步骤](#安装步骤)
```

**同学**：这样可以了！但我的标题有空格怎么办？比如 `### 1.1 Markdown介绍`

**管理员**：空格也原样保留，不用替换：
```markdown
[跳转](#1.1 Markdown介绍)
```

**同学**：哦哦！原来不用替换空格！

---

### 8.3 图片显示不出来？

**同学**：图片显示红叉！路径我检查了好几遍是对的！

**管理员**：用的是绝对路径还是相对路径？

**同学**：`D:\\Projects\\image.png`

**管理员**：两个问题：
1. Markdown 里路径分隔符用 `/`，不是 `\`
2. 最好用相对路径，比如 `./images/image.png`

改成这样试试：
```markdown
![描述](./images/image.png)
```

**同学**：可以了！

---

### 8.4 代码块没有高亮？

**同学**：我的代码块全是灰色的，没有颜色！

**管理员**：你指定语言了吗？

**同学**：什么语言？

**管理员**：代码块开头要加语言标识，比如：
```markdown
def hello():
    print("Hello")
```

**同学**：原来要加 `python`！

---

### 8.5 目录怎么自动生成？

**同学**：我看别人的文档有自动目录，怎么弄的？

**管理员**：在文档里任意位置插入：
```markdown
[TOC]
```

Typora 会自动根据你的标题生成目录。

**同学**：就这么简单？！

**管理员**：就这么简单。

## 9 检查清单

发布文档前检查：

- [ ] 无 `>` 引用块做信息展示
- [ ] 无行尾双空格
- [ ] 无禁止的 HTML 标签（`<details>`、`<div style>`、`<script>` 等）
- [ ] 允许的简单 HTML 正确使用（`<u>`、`<font>`、`<br>`、`<a id>` 等）
- [ ] 表格列数一致
- [ ] 代码块指定语言
- [ ] 链接可点击
- [ ] 图片路径正确（使用相对路径）
- [ ] 页内跳转有效（标题原样复制，空格不替换）
- [ ] 目录与标题对应
