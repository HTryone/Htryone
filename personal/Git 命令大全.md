# 个人使用

------

# 🧭 一、先记住唯一核心

```text
改文件 → add → commit
```

对应：

```text
工作区（你改的）
↓ git add
暂存区
↓ git commit
历史版本
```

👉 你现在的核心：**控制"工作区长什么样"**

|                        | 看历史           | git log --oneline                            |                                                              |
|                        | 看状态           | git status                                   |                                                              |
|                        | 看分支           | git branch                                   |                                                              |
|                        |                  |                                              |                                                              |
|                        |                  |                                              |                                                              |
| **恢复修改及回退版本** |                  |                                              |                                                              |
|                        | 改乱了（没 add） | git restore .                                | 没有暂存的情况下恢复，**新建文件不在操作范围**               |

> 如有疑问，先用 `git status` 和 `git log --oneline` 确认当前状态，再执行改写历史的操作。

## git reset --hard🚑 后悔药

| 做什么       | 命令                                    | 说明                                                         |
| ------------ | --------------------------------------- | ------------------------------------------------------------ |
| 查看操作记录 | `git reflog`                            | 找你想回去的那个版本号，从上往下看。或者使用sourcetree看更直观 |
| 恢复         | `git reset --hard 某个ID`是commit的hash | 回去                                                         |

> ❗ reset 会删历史——👉 该版本之后的所有提交记录会永久消失。
>
> ⚠️ **不可逆操作**——如果不确定，先用 `git reflog` 保存好 ID 再操作。回滚时工作区最干净不会混淆文件。其他命令推荐清空工作区再执行。备份一遍准没错！

**💡 最省事用法：** `reset --hard` 到某版本 → 随便折腾 → `reflog` 找到之前的 ID → `reset --hard` 回来。不用操心删文件，`--hard` 会自动把工作区彻底变成目标版本。

| 看分支   | `git branch`        |
| 切换分支 | `git switch main`   |
| 创建分支 | `git switch -c dev` |

## 🌳 常见开发

| 场景             | 命令                                           |
| ---------------- | ---------------------------------------------- |
| 在主线开发       | `git switch main` → `git add .` → `git commit` |
| 基于旧版本开分支 | `git checkout 版本ID` → `git switch -c new`    |

# ⚠️ 三、常见困惑

## ❗ checkout 会"离开分支"

`git checkout 版本ID` 会进入 **detached HEAD（游离状态）**——脱离正常分支，站在某个历史版本上。**可以看，但不要在这里写代码**（写了很难保存）。用 `git switch main` 回来。

## ❗ commit 只提交 add 过的内容

没 `git add` 的修改不会进入版本。

## 🧭 确认"我在哪"

| 你看到什么          | 说明                               | 怎么回来            |
| ------------------- | ---------------------------------- | ------------------- |
| `On branch main`     | 正常在主分支上                     | 不用回来            |
| `HEAD detached at x` | 停在某个历史版本中（游离状态）     | `git switch main`   |
